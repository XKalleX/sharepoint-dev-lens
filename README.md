# SharePoint Dev Lens

**The practical SharePoint inspector for developers, consultants and Microsoft 365 admins.**

A browser extension for **Chrome** and **Edge** that makes hidden SharePoint technical details visible right in the browser — list GUIDs, field InternalNames, content types, permissions, REST endpoints and governance warnings.

> Not affiliated with or endorsed by Microsoft Corporation.

---

## Features (v1.0)

| Tab | What you get |
|-----|-------------|
| 🏠 **Site** | Web ID, template, LCID, Hub Site status, REST endpoints, admin links, governance warnings |
| 📋 **Lists** | All lists & libraries with GUIDs, BaseTemplate, item counts, versioning status, copy buttons |
| 🔍 **Fields** | InternalName, StaticName, Field ID, type, required/indexed flags, REST/PnPjs/Power Automate snippets |
| 🔗 **Links** | Developer quick links to `/_layouts/15/`, REST API + custom URL opener |
| ⬇ **Export** | Download/copy as JSON, Markdown or CSV for documentation |

**Privacy:** No data collection. All processing is local in your browser using your existing SharePoint session. No OAuth, no external servers.

## Quick Start

### 1. Generate icons (once, requires Node.js)

```
node scripts\generate-icons.js
```

### 2. Build both packages

```
.\build.ps1
```

Or double-click `build.bat`.

Output:
```
dist\
├── chrome\   ← load as unpacked extension in Chrome
└── edge\     ← load as unpacked extension in Edge
```

### 3. Load in your browser

**Chrome**
1. Open `chrome://extensions/`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked** → select `dist\chrome\`

**Edge**
1. Open `edge://extensions/`
2. Enable **Developer mode**
3. Click **Load unpacked** → select `dist\edge\`

## Project Structure

```
sharepoint-dev-lens/
├── src/                        Source files (same for both browsers)
│   ├── background.js           MV3 service worker – opens side panel on click
│   ├── content.js              Content script – SharePoint REST API + governance
│   ├── popup.html / popup.js   Toolbar popup
│   ├── sidepanel.html          Side panel shell
│   ├── css/styles.css          Microsoft Fluent Design-inspired stylesheet
│   ├── js/
│   │   ├── sidepanel.js        Tab UI, state, rendering (vanilla JS)
│   │   └── modules/
│   │       ├── messaging.js    chrome.tabs.sendMessage helpers
│   │       ├── snippets.js     REST / PnPjs / Power Automate snippet builder
│   │       ├── export.js       JSON / Markdown / CSV export
│   │       └── utils.js        Copy, escape, format helpers
│   └── icons/                  Generated by scripts/generate-icons.js
├── manifests/
│   ├── manifest.chrome.json
│   └── manifest.edge.json
├── scripts/
│   └── generate-icons.js       Pure Node.js icon generator (no npm needed)
├── store/
│   ├── chrome/listing.md       Chrome Web Store listing text
│   └── edge/listing.md         Edge Partner Center listing text
├── build.ps1                   Build script (PowerShell)
├── build.bat                   Wrapper for build.ps1
└── build.sh                    Build script (macOS / Linux)
```

**Data flow:**
1. Popup or Side Panel sends a typed message to the content script
2. Content script calls `/_api/…` using the user's authenticated browser session (same-origin cookies, no OAuth needed)
3. Data is returned and rendered in the vanilla-JS UI

## Requirements

- Node.js (any recent version) – only needed once to generate icons
- Chrome 116+ or Edge with MV3 support

## No npm, no build tools, no framework

The extension is plain HTML, CSS and modern JavaScript (ES Modules). No transpilation, no bundler. Load `dist\chrome\` or `dist\edge\` directly.
